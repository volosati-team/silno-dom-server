# Zone panel + ctrl-btn UI compact mode — 2026-05-24

План правок ctrl-btn (кнопки-toggle Споты/Гирлянда/Ворота/etc) и
layout zone-clip ↔ media-clip. Записан Андреем 2026-05-24 01:53 MSK,
будет реализовываться следующей сессией.

## Правки на ctrl-btn

1. **Убрать строку текста статуса.** Сейчас под основным текстом
   toggle (`<span class="cl">Споты</span>`) висит `dev-row` с
   `dev-dot` + `dev-lbl` («вкл» / «выкл» / «—»). Лейбл «вкл/выкл»
   **выкинуть** полностью.

2. **Status dot — в верхний левый угол.** Точка (`dev-dot`) диаметром
   5–10px, position absolute, top/left по смещению равному диаметру
   (т.е. если dot=8px, top=8px / left=8px). Без подписи рядом, без
   подписи внизу.

3. **Уменьшить высоту кнопки.** Сейчас `.ctrl-btn` ближе к квадрату
   (потому что внутри toggle-switch + cl-text + dev-row). После выкида
   dev-row кнопка автоматически станет короче. Дополнительно сделать
   её более **продольной** (горизонтально вытянутой) — например `height`
   фиксированный ~56px, `min-width: 140px`.

## Layout: media-panel + zone visible одновременно

Сейчас `openMediaPanel()` вызывает `closeZone()` — кнопки света
исчезают при открытом плеере. Андрей хочет:

> «Когда у нас плеер открыт, мы видим всё равно кнопки, которые относятся
> к этому дому. Если их будет больше, если там будут ещё какие-то поля,
> какие-то виджеты, они будут опускаться. Сейчас мне важно, чтобы при
> работающем плеере я увидел кнопки Спот, Гирлянда и так далее.»

> «Media Widget поднимается на высоту минус хедер, минус табы и минус
> кнопки. Минус кнопки 1 ряд.»

Математика:

```
media-clip.height = 100vh - var(--hh) - var(--zh) - <1 row ctrl-btn>
```

где `--hh` — высота `#header`, `--zh` — высота `#zone-tabs`, `1 row` —
высота **одной** строки ctrl-btn (после правок выше, ~64–72px с padding).

То есть:
- При открытом media-panel `zone-clip` **остаётся открытым** на
  одну строку кнопок текущей активной зоны.
- Кнопки видны параллельно с плеером.
- При закрытом media-panel `zone-clip` может развернуться на полный
  размер для нескольких рядов (как сейчас).

Если кнопок больше чем умещается в 1 row — должна работать горизонтальная
прокрутка / wrap. Андрей: «если будут ещё какие-то поля, какие-то виджеты,
они будут опускаться» — то есть **новые виджеты идут вниз**, основной ряд
остаётся видимым.

## Реализация (план для следующей сессии)

1. CSS `.ctrl-btn`: убрать `.dev-row`, position-absolute `.dev-dot`
   в top-left, фиксированная высота `~56px`.
2. CSS `#media-clip.open`: высота `calc(100vh - var(--hh) - var(--zh) - <row-h>)`.
3. JS `openMediaPanel()`: **не** вызывать `closeZone()` если активная
   зона есть. Просто оставить как было.
4. CSS `#zone-clip.open ~ #media-clip.open`: media-clip начинается
   ниже первого ряда кнопок.
